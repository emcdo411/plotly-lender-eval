The Method

Three Gates. One Standard.

Every finding in this repo runs through the same sequential diagnostic before it's allowed to ship. No layer gets skipped, and no tool gets named until the first two clear.

01 MOC v4.9

Ambient Signal Read

Reads the market-wide signal, scores opportunity fit against the actual use case, and names the hard blockers up front. Interest in a capability and readiness to deploy it are treated as two different timelines.

02 FCPB v2.9

Subtraction Audit

The Fraser Gate asks one question before anything else: does this tool remove complexity from the stack, or add to it? Unscoreable until the incumbent BI environment is mapped, so it doesn't get scored early.

03 FCI v2.3

Positioning Gate

Keeps the output problem-first, not vendor-first. Every claim is tagged by confidence level instead of stated as settled fact, because that's the moment a proof-of-work document loses the room.

Signature System

Evidence Facts

Every factual claim in this repo carries one of four tags. Nothing ships labeled as certain when it isn't.

This is the same discipline a compliance reviewer applies to a vendor pitch, just run against our own document first. If a claim can't be traced to a public source, it doesn't get to sound like one.

  • Confirmed capability claims cite the company's own statements or press.
  • Inferred claims are marked as inference, not fact.
  • Unconfirmed ideas are labeled a hypothesis, not a finding.
  • Absence of evidence is reported as exactly that, not treated as proof something doesn't exist.
