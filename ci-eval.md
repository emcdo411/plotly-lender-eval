## 1. What Plotly Skills Actually Is

Plotly Studio has released a native **Skills** capability: reusable, saved prompts invoked via slash command inside a session. Skills can:

- Enforce a specific output type (app, chart, or report)
- Pre-load data-source connection instructions to reduce setup time across sessions
- Apply consistent stylistic guidelines to a data app
- Standardize recurring analysis workflows across projects and teams
- Be built custom per workspace (name it, write the instructions, it becomes a slash command)

This sits inside Plotly's broader Studio/Dash Enterprise agentic push and is distinct from third-party "Plotly skill" packages built by the open-source community for coding agents like Claude Code — those are chart-generation reference libraries, not the same product surface.

**Confidence tagging:**
- Capability description above: **verified against Plotly's own documentation**
- Fit for any specific regulated-finserv client integrations function: **hypothesis, pending internal stack visibility**

This is a legitimate product direction, not hype.

| Factor | Assessment |
|---|---|

| Solution-Landscape Redundancy | **Unknown** — depends entirely on incumbent BI stack, not yet visible |
| Data Leverage Score | **Unknown** — depends on what data is permitted to touch a third-party visualization layer |

**Hard blocker:** In a regulated title/mortgage servicing environment, any new tool touching client or loan-level data requires security and compliance review before it becomes a live option. This is a sequencing constraint, not a merit judgment — capability interest and deployment readiness are two different timelines.

## 3. Layer 2 — Complexity & Organizational Fit (Fractional CXO Practice Builder v2.9)

**Subtraction audit (Fraser Gate):** Whether Plotly Skills reduces or adds to the organization's complexity stack cannot be scored without first mapping the incumbent reporting/BI environment. Proposing a net-new vendor before that mapping is complete is complexity addition by default.

**Complexity Efficiency Score:** Not yet scoreable — requires a baseline comparison against whatever ServiceLink currently runs for integration reporting and dashboards.

**Recommended sequencing:**
1. First 30 days: observe and document the existing reporting/BI stack in the Client Integrations function
2. Days 30–60: identify specific, named gaps the current stack doesn't close
3. Days 60–90: if a genuine gap exists, evaluate 2–3 candidate tools against it, Plotly among them, on documented criteria

This sequencing protects the credibility of any eventual recommendation — a solution proposed before the problem is documented reads as vendor-first, not problem-first.
