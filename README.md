## Overview

This repository is a working evaluation of Plotly Studio's native **Skills** capability, scoped specifically to a regulated client integrations environment in the title and mortgage servicing industry. It was built as a pre-deployment capability assessment: no live implementation, no non-public data, and no procurement decision implied.

Every claim in this repo is confidence-tagged rather than presented as settled fact. That discipline is the point. A recommendation proposed before the problem is documented reads as vendor-first, not problem-first, and this repo is built to avoid that trap.

## Methodology

Both documents route through a sequential three-layer diagnostic:

1. **Marketing Ops Catalyst v4.9** — ambient signal read (ASIL), structured signal translation (DIL), and competitive benchmarking (ESIL/AWSS)
2. **Fractional CXO Practice Builder v2.9** — subtraction audit (Fraser Gate) to test whether a proposed tool reduces or adds to the existing complexity stack
3. **Founder CEO Intelligence v2.3** — positioning and credibility gate to keep any recommendation problem-first rather than tool-first

## Evidence & Claim-Tagging Standard

Every factual claim in this repo carries one of four tags:

| Tag | Meaning |
| --- | --- |
| `[PUBLIC-VERIFIED]` | Confirmed via company statements, press, or public filings |
| `[INFERRED-PUBLIC]` | Reasonable inference drawn from verified public signals |
| `[HYPOTHESIS]` | Plausible but unconfirmed |
| `[ABSENCE OF PUBLIC EVIDENCE]` | Nothing found publicly; this does not mean the capability doesn't exist internally, only that it isn't publicly signaled |

## Status & Next Steps

This evaluation is a working document, not a procurement recommendation. It does not commit any organization to a vendor decision. The intended next step is a 30 to 60 day internal stack-mapping window, after which a short list of 2 to 3 candidate tools (Plotly among them) can be evaluated against documented, named gaps rather than assumed ones.
